Run a CKB Node

To get started on your journey with CKB, the first thing you can try is to run a CKB node yourself.

If you are not familiar with the concepts of node and mining yet, here is a document you can learn from.

CKB testnet will be launched very soon. When it does, your node can join the testnet to be a part of it, or you can just interact with one of the existing testnet nodes without running your own node.

Get CKB

To get the CKB client software, you can choose to download the released binary directly, or build it from the source code, or run it in a docker.

Download from Release

Dependencies

For Linux user, you need to install libssl dynamic libraries before using the released binary.

sudo apt-get install -y libssl1.0.0

Download

You can find the CKB releases on GitHub Releases. Just choose the file to download according to your operating system. (You can also find nightly build versions from the releases of the ckb-builds repo)

After it is downloaded, you need to move it to a PATH directory:

ln -snf "$(pwd)/ckb" /usr/local/bin/ckb

Then check if it works with:

ckb --version

Compile from Source

Dependencies

CKB is developed with Rust. To get Rust and its tools, please refer to the rustup website for how to install rustup.

Then install dependencies with the following command:

Ubuntu and Debian
Arch Linux
macOS
sudo apt-get install -y git gcc libc6-dev pkg-config libssl-dev libclang-dev clang
sudo pacman -Sy git gcc pkgconf clang
brew install autoconf libtool

Build

Here we build from the source code on Master branch, which is the latest release version. For other versions and branches, please refer CKB repo for more information.

Get the source code:

git clone https://github.com/nervosnetwork/ckb.git
cd ckb
git checkout master

Install Rust (currently, CKB is mainly tested with version stable-1.34.1):

rustup override set 1.34.1

Build it:

Make Build

You will find the built executable binary in target/release/ckb folder.

Then you need to move it to a PATH directory:

ln -snf "$(pwd)/ckb" /usr/local/bin/ckb

Then check if it works with:

ckb --version

Use Docker

You can also use Docker to run your CKB node. For the detailed guidance, please refer to this document on GitHub.

Run CKB

First, you need to have a workshop folder to run CKB:

mkdir ckb-dev
cd ckb-dev

Then generate default configuration files:

ckb init

See here for more details about how to configure CKB.

Then you can start the node:

ckb run
